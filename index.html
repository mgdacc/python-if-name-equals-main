Cheat Sheet de Python

if __name__ == "__main__"

1 / 10

El Problema Central

Un archivo Python puede usarse de dos maneras:

1. Ejecutándolo directamente.

2. Importándolo en otro archivo.

Necesitamos una forma de que el código se comporte diferente en cada caso.

2 / 10

La Pieza Clave: __name__

Python crea una variable especial llamada __name__ en cada script.

Su valor cambia dependiendo de cómo se usa el archivo, permitiéndonos detectar el modo de ejecución.

3 / 10

El Comportamiento de __name__

Modo de Ejecución Valor de __name__
Directa:
$ python mi_script.py		 "__main__"
Indirecta:
import mi_script		 "mi_script"
4 / 10

Caso de Estudio: Un Módulo Simple

Imaginemos un módulo calculadora.py. Contiene una función y un código de ejemplo que se ejecuta siempre.

# calculadora.py

def sumar(a, b):
    return a + b

# Este código se ejecuta SIEMPRE
print("Módulo calculadora cargado.")
resultado_ejemplo = sumar(5, 3)
print(f"Ejemplo: 5 + 3 = {resultado_ejemplo}")
5 / 10

El Efecto Secundario al Importar

Ahora, un archivo principal.py importa calculadora para usar su función.

# principal.py
import calculadora

total = calculadora.sumar(10, 20)
print(f"El total es: {total}")

Problema: El código de ejemplo de calculadora.py se ejecuta, ensuciando la salida.

6 / 10

La Salida Indeseada

Al ejecutar principal.py, vemos esto en la terminal:

$ python principal.py
Módulo calculadora cargado.
Ejemplo: 5 + 3 = 8
El total es: 30

Queremos que solo se muestre la última línea.

7 / 10

La Solución: Proteger el Código

Movemos el código de ejemplo dentro del bloque if __name__ == "__main__" en calculadora.py.

# calculadora.py
def sumar(a, b):
    return a + b

# Este código ahora está protegido
if __name__ == "__main__":
    print("Módulo ejecutado directamente.")
    resultado_ejemplo = sumar(5, 3)
    print(f"Ejemplo: 5 + 3 = {resultado_ejemplo}")
8 / 10

El Resultado Final y Limpio

Ahora, al ejecutar principal.py de nuevo, la condición if en calculadora.py es falsa (porque __name__ es "calculadora" debido a que llama (o ejecuta de manera indirecta) a calculadora.py).

$ python principal.py
El total es: 30

¡Perfecto! El módulo es ahora 100% reutilizable y sin efectos secundarios.

9 / 10

Conclusión

Usa if __name__ == "__main__": para:

1. Crear módulos reutilizables.

2. Separar la lógica principal de las definiciones de funciones y clases.

3. Incluir pruebas o demos que solo se ejecuten cuando corres el archivo directamente.

10 / 10
